## 1. 项目概述

### 1.1 项目简介

`owlapi-inference-demo` 是一个基于Spring Boot的知识图谱管理平台，核心功能是将关系型数据库的Schema自动转换为OWL本体和OBDA映射文件，并支持与GraphDB的无缝集成。

### 1.2 核心能力

- **Schema扫描**: 连接关系数据库，自动扫描表结构和外键关系
- **本体生成**: 根据数据库Schema自动生成OWL本体(OWL Ontology)
- **映射生成**: 生成OBDA映射文件，实现关系数据到RDF的虚拟映射
- **GraphDB管理**: 提供完整的GraphDB仓库管理功能
- **SPARQL查询**: 支持在GraphDB上执行各种SPARQL查询

### 1.3 启动方式

```bash
# 进入项目目录
cd c:\Users\tzp\owlapi-inference-demo

# 编译并启动
mvn spring-boot:run

# 或打包后运行
mvn clean package
java -jar target/owlapi-inference-demo-1.0-SNAPSHOT.jar
```

启动后访问: http://localhost:8083

## 4. 核心功能模块

### 4.1 Schema扫描与本体生成

**流程图:**

```
关系数据库 ──Schema扫描──> 表结构信息 ──OWL生成器──> OWL Ontology
                                │
                                └──OBDA生成器──> OBDA Mappings
                                                       │
                                                       ▼
                                                  GraphDB
```

**OWL生成规则:**

| 数据库元素 | OWL元素 |
|-----------|---------|
| 表(Table) | OWL Class |
| 列(Column) | OWL DataProperty |
| 外键(FK) | OWL ObjectProperty |
| 数据类型 | XSD Datatype |

**命名规范:**

- 类IRI: `{baseIri}{normalize(tableName)}`
- 数据属性IRI: `{baseIri}{normalize(table_column)}`
- 对象属性IRI: `{baseIri}{normalize(ref_fkTable_to_pkTable)}`

### 4.2 OBDA映射说明

OBDA (Ontology-Based Data Access) 映射文件实现虚拟RDF视图:

```obda
[PrefixDeclaration]
:           http://example.com/ontology#
obda:        https://w3id.org/obda/vocabulary#

[MappingDeclaration] @collection [[

mappingId    MAPPING-ID1
target      :Patient/{patient_id} a :Patient ;
             :Patient_name {name}^^xsd:string .
source      SELECT * FROM `Patient`

]]
```

### 5.2 SPARQL查询API

#### 执行SELECT查询

```
POST /api/graphdb/repositories/{repoId}/sparql
Content-Type: application/x-www-form-urlencoded

query=SELECT ?s ?p ?o WHERE { ?s ?p ?o } LIMIT 10
```

**响应示例:**
```json
{
  "head": {
    "vars": ["s", "p", "o"]
  },
  "results": {
    "bindings": [
      {
        "s": {"type": "uri", "value": "http://example.com/Patient/1"},
        "p": {"type": "uri", "value": "http://www.w3.org/1999/02/22-rdf-syntax-ns#type"},
        "o": {"type": "uri", "value": "http://example.com/ontology#Patient"}
      }
    ]
  }
}
```

#### 执行CONSTRUCT查询

```
POST /api/graphdb/repositories/{repoId}/sparql/construct
Content-Type: application/x-www-form-urlencoded

query=CONSTRUCT { ?s ?p ?o } WHERE { ?s ?p ?o } LIMIT 10
format=turtle
```

#### 执行ASK查询

```
POST /api/graphdb/repositories/{repoId}/sparql/ask
Content-Type: application/x-www-form-urlencoded

query=ASK { ?s a <http://example.com/ontology#Patient> }
```

## 6. Java API参考

### 6.1 SchemaToOwlGenerator

OWL本体生成器，用于将数据库Schema转换为OWL Ontology。

**Maven依赖:** (已包含在项目中)

```xml
<dependency>
    <groupId>net.sourceforge.owlapi</groupId>
    <artifactId>owlapi-distribution</artifactId>
    <version>5.1.9</version>
</dependency>
```

**核心方法:**

```java
import com.example.owlapi.SchemaToOwlGenerator;
import com.example.owlapi.SchemaGeneratorArgs;
import com.example.owlapi.SchemaGenerationResult;

// 创建配置参数
SchemaGeneratorArgs args = new SchemaGeneratorArgs();
args.jdbcUrl = "jdbc:mysql://localhost:3306/medical_db";
args.user = "root";
args.password = "123456";
args.driverClass = "com.mysql.cj.jdbc.Driver";
args.catalog = "medical_db";
args.schema = "";
args.tablePattern = "%";
args.baseIri = "http://example.com/ontology#";
args.output = "schema-auto.owl";
args.includeTables = new HashSet<>();
args.excludeTables = new HashSet<>();

// 执行生成
SchemaGenerationResult result = SchemaToOwlGenerator.generate(args);

// 获取结果
System.out.println("Tables: " + result.tableCount);
System.out.println("DataProperties: " + result.dataPropertyCount);
System.out.println("ObjectProperties: " + result.objectPropertyCount);
System.out.println("Output: " + result.outputPath);
```

**数据类型映射表:**

| SQL数据类型 | XSD数据类型 |
|------------|-------------|
| INTEGER, SMALLINT, TINYINT | xsd:integer |
| BIGINT | xsd:long |
| FLOAT, DOUBLE, REAL | xsd:double |
| DECIMAL, NUMERIC | xsd:decimal |
| BOOLEAN, BIT | xsd:boolean |
| DATE | xsd:date |
| TIME | xsd:time |
| TIMESTAMP | xsd:dateTime |
| 其他 | xsd:string |

### 6.4 GraphDbImportService

GraphDB数据导入服务。

**核心方法:**

```java
import com.example.owlapi.graphdb.GraphDbImportService;
import org.eclipse.rdf4j.rio.RDFFormat;
import java.io.FileInputStream;

@Autowired
private GraphDbImportService importService;

// 导入OWL文件到指定仓库
FileInputStream input = new FileInputStream("ontology.owl");
importService.importOwlFileToRepository("test", input, RDFFormat.RDFXML);

// 导入RDF数据
FileInputStream rdfInput = new FileInputStream("data.ttl");
importService.importRdfToRepository("test", rdfInput, RDFFormat.TURTLE);

// 清空仓库
importService.clearRepository("test");

// 获取仓库统计
Map<String, Object> stats = importService.getRepositoryStats("test");
// stats包含: statementCount, isEmpty, namespaces
```

**支持的RDF格式:**

| 格式 | RDFFormat常量 |
|------|--------------|
| RDF/XML | `RDFFormat.RDFXML` |
| Turtle | `RDFFormat.TURTLE` |
| N-Triples | `RDFFormat.NTRIPLES` |
| N3 | `RDFFormat.N3` |

## 9. GraphDB对接指南

### 9.1 GraphDB REST API

项目通过RDF4J库对接GraphDB REST API:

**HTTP Repository:**

```java
import org.eclipse.rdf4j.repository.http.HTTPRepository;
import org.eclipse.rdf4j.repository.RepositoryConnection;
import org.eclipse.rdf4j.query.TupleQuery;
import org.eclipse.rdf4j.query.QueryLanguage;
import org.eclipse.rdf4j.query.TupleQueryResult;

// 创建连接
String graphDbUrl = "http://localhost:7200";
String repositoryId = "test";
HTTPRepository repository = new HTTPRepository(graphDbUrl, repositoryId);
repository.init();

// 执行查询
try (RepositoryConnection conn = repository.getConnection()) {
    TupleQuery query = conn.prepareTupleQuery(
        QueryLanguage.SPARQL,
        "SELECT * WHERE { ?s ?p ?o } LIMIT 10"
    );
    
    try (TupleQueryResult result = query.evaluate()) {
        while (result.hasNext()) {
            System.out.println(result.next());
        }
    }
}

// 关闭连接
repository.shutDown();
```

## 附录: 常见问题

### Q1: 启动时数据库连接失败

**检查:**
1. MySQL服务是否启动
2. 数据库URL是否正确
3. 用户名密码是否正确
4. 防火墙是否允许3306端口

### Q2: GraphDB仓库连接失败

**检查:**
1. GraphDB服务是否启动
2. GraphDB URL是否正确
3. 仓库ID是否存在

### Q3: OWL和OBDA命名不一致

**原因:** OWL生成器和OBDA生成器使用相同的normalize函数，确保命名一致。

### Q4: SPARQL查询无结果

**检查:**
1. 仓库是否为空
2. 查询语句是否正确
3. 前缀是否正确声明
